System

  • 3 independent microcontroller-based architecture (RPI2040, RPI2040-ZERO, ESP32)
  • Primary mission: temperature and pressure measurement with redundant sensors
  • Secondary missions:
    • Life-support cabin (temperature and pressure regulation)
    • Cloud observation with AI-based classification
    • Redundant primary mission using Kalman filter
  • 868 MHz LoRa communication (868.1 and 868.9 MHz, software adjustable)
  • Onboard data storage on SD card (~277 MB maximum data)
  • Designed for minimum 4 hours of active operation
  • Total system mass: 345g
  • Total cost: 483,5 Euro

Mechanics

  • Outer shell: 3D printed tough BLK resin
  • Internal structural elements: 6082 aluminum alloy
  • Survival chamber designed with reinforced wall thickness for pressure resistance
  • Pressurized air tank with safety valve
  • 3 O-ring sealing system:
    • Pressure chamber–cover
    • Cover–survival chamber
    • Survival chamber–Peltier housing
  • Peltier cooling element in copper housing with upper and lower heat sinks
  • Tested for 20G acceleration and 50N load
  • Parachute attached via crossed metal cord structure and swivel ring
  • Two full CanSat units prepared for finals (shared metal components)

Software

  • Programming languages: C++ (MCUs), Python (RPI2040)
  • Development environments: VS Code, Arduino IDE
  • Core functions:
    • Sensor data acquisition
    • Data formatting (base-61 encoding)
    • Telemetry transmission
    • Flight phase detection (launch, descent, landing)
  • Automatic phase switching based on acceleration and pressure data
  • Fault tolerance:
    • 2× retry for sensor initialization
    • Graceful degradation mode
    • Safe default state (valve closed, Peltier off)
  • Kalman filter for data fusion
  • AI-based cloud classification using 96×96 grayscale image

Telemetry

  • LoRa communication in the 868 MHz ISM band
  • Minimum data rate: 255 bytes/s
  • Two independent LoRa channels for redundancy
  • Base-61 compressed data transmission (Z delimiter)
  • GPS coordinates transmitted every 10 seconds after landing
  • Images stored on SD card, one image transmitted to ground
  • Ground station:
    • Two receiving units
    • SY-910 11.5 dB Yagi Antenna
    • Gamma matching for SWR optimization
  • Data exported in CSV format
  • Image processing using Python, NumPy, and OpenCV

Electronics

  • 3× four-layer PCBs, 60 mm diameter
  • Power traces minimum 4 mm width
  • Main components:
    • RPI2040
    • RPI2040-ZERO
    • ESP32
    • BMP390, BMP280
    • MAX6675 K-type sensor
    • MPU6500
    • 2× LoRa modules
    • GPS module
  • Dedicated communication board
  • Dedicated primary mission board
  • Dedicated secondary mission board
  • LiPo batteries with individual cell protection
  • Master power switch
  • Total power consumption: ~2.7 W

Recovery

  • Cross-shaped ripstop nylon parachute
  • Descent rate: 7.8m/s
  • 8 micro cords connected to a central swivel ring
  • Stable, anti-twist descent configuration
  • Estimated descent time from ~1 km altitude
  • GPS-based recovery system
  • Position transmitted every 10 seconds after landing
